If you're diving into open-source chip design on Arch Linux, you'll quickly realize that managing analog tools (Magic, Ngspice, Xschem) alongside digital tools (Yosys, OpenROAD) can turn into dependency chaos. Different tools demand conflicting library versions, and your system becomes a fragile house of cards.

**The solution? Isolated Docker containers.**

In this comprehensive guide, I'll show you how to set up two professional-grade VLSI environments:
1. **Analog Container** (IIC-OSIC-TOOLS) for schematic capture, SPICE simulation, and layout
2. **Digital Container** (LibreLane) for RTL-to-GDS flow with Nix-based reproducibility
3. **Shared PDK Repository** (Sky130) to avoid redundant 5GB+ downloads

This is the exact workflow I use for neuromorphic hardware research at SMVDU.

Why This Setup Rocks
======

**Isolation:** Analog and digital tools never conflict. Each lives in its own container.

**Persistence:** Your designs live on your host filesystem (`~/vlsi/`), not inside containers.

**Efficiency:** One centralized PDK folder serves both containers.

**Reproducibility:** The Nix-based LibreLane container ensures your digital flow works identically on any machine.

**Professional:** This is how modern ASIC teams work—containerized, version-controlled, scriptable.

Phase 1: System Prerequisites
======

### Install Docker and Essential Tools

First, ensure Docker is installed and your user has permission to run it without `sudo`.

```bash
# Update system
sudo pacman -Syu

# Install Docker, Git, Python, and build tools
sudo pacman -S docker git base-devel python pip

# Start and enable Docker service
sudo systemctl start docker
sudo systemctl enable docker

# Add your user to the docker group (CRITICAL for file permissions)
sudo usermod -aG docker $USER

# Apply group changes (or log out and back in)
newgrp docker

# Verify Docker works
docker run hello-world

# Create the main workspace
mkdir -p ~/vlsi

# Subdirectories for each component
mkdir -p ~/vlsi/pdk_root           # Shared PDK storage
mkdir -p ~/vlsi/analog_designs     # Analog schematics/layouts
mkdir -p ~/vlsi/digital_designs    # Digital RTL projects
mkdir -p ~/vlsi/librelane_flow     # LibreLane workspace

# Add to ~/zshrc
echo 'export PDK_ROOT=$HOME/vlsi/pdk_root' >> ~/.zshrc
echo 'export PDK=sky130A' >> ~/.zshrc
echo 'export STD_CELL_LIBRARY=sky130_fd_sc_hd' >> ~/.zshrc

# Apply immediately
source ~/.zshrc

# Verify
echo $PDK_ROOT
```

PHASE 2:INSTALLING THE SKY130 PDK with Volare
======

The Process Design kit contaisn all the process-specific data :transistor models , design rules,standard cells and I/O libraries. 

```bash

Lets install Volare

# Install the PDK manager
pip install --upgrade --no-cache-dir volare

# Add pip binaries to PATH if needed
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# Verify installation
volare --version
 
```

# Download and Enable Sky130PDK
# Set the target directory
export PDK_ROOT=$HOME/vlsi/pdk_root

# List available versions (optional, to see what's available)
volare ls-remote --pdk sky130

# Download and enable the default Sky130 version
# This takes 5-15 minutes depending on internet speed
volare enable --pdk sky130
